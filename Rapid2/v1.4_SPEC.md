## 1. Goal

v1.4 replaces v1.3's monolithic regime/QF strategy with a **Core + Satellite** split on a $90 account. Design goals, in priority order:
1. **Capital preservation** — survive long enough to compound.
2. **Beat passive BTC** on a risk-adjusted basis (Sharpe > 1.0 target).
3. **Minimize complexity** — one active strategy, not three.
4. **Paper-trade-ready by end of build** — no live deploy in this cycle.

The design is grounded in evidence collected 2026-04-20:
- Fear-biased DCA returned ~1,145% from 2018–2025 (SpotedCrypto).
- 92% of active crypto traders underperform passive (Binance Research 2024).
- Mean-reversion with a trend filter is one of the few retail-viable active edges.

- **Core** = Enhanced Fear-DCA. 70% of account. Weekly auto-buys BTC+ETH, boosted when F&G low.
- **Satellite** = Mean-reversion agent on BTC/USD. 30% of account. One active strategy, one coin, one open position at a time.
- **Capital module** = Survival envelope. Pauses satellite below a threshold, alerts at floor.
- **Strategy orchestrator** = Thin layer that calls the satellite agent, applies the circuit breaker, returns a decision.

**Explicitly out of scope for v1.4:**
- Multi-agent orchestration (no breakout, no range, no momentum agents)
- Conviction-based position sizing
- Regime switching via F&G *for trade selection* (F&G is used ONLY for DCA sizing and DUST filtering)
- Meme coin / DUST trading
- LunarCrush / CryptoCompare social signals
- Trailing stops / TP ladders

## 4. Agent Interface (agents/base.py)

```python
from dataclasses import dataclass
from typing import Any, Literal

@dataclass(frozen=True)
class AgentContext:
    """Everything an agent needs to make a decision. Immutable."""
    exchange:                   Any         # ccxt exchange instance (Kraken)
    symbol:                     str         # e.g. "BTC/USD"
    current_price:              float
    account_value:              float       # Total account USD
    satellite_capital:          float       # USD allocated to the satellite tier
    open_satellite_position:    dict | None # {"entry_price": float, "entry_ts": float, "size_usd": float} or None
    now_ts:                     float       # Unix timestamp (seconds)
    btc_above_200ma:            bool        # Trend gate — supplied by orchestrator

@dataclass(frozen=True)
class AgentSignal:
    """Agent's decision. Pure output — orchestrator decides what to do with it."""
    action:               Literal["buy", "sell", "hold"]
    confidence:           float       # 0.0–1.0 (informational; NOT used for sizing in v1.4)
    stop_pct:             float       # e.g. 0.04 for 4%
    target_pct:           float       # e.g. 0.08 for 8%
    reason:               str         # Human-readable
    expected_hold_hours:  float       # Rough estimate

def evaluate(ctx: AgentContext) -> AgentSignal: ...
```

Every agent is **one pure function**: `evaluate(ctx) -> AgentSignal`. No side effects, no network calls outside what ctx provides for OHLCV fetching.

## 5. Core — Enhanced Fear-DCA (core/dca.py)

**Behavior:** Weekly buys of BTC/USD and ETH/USD. Skip buys when BTC is far above its 200-day MA (top-filter). Boost buy size when F&G is low.

### Public function

```python
from dataclasses import dataclass

@dataclass(frozen=True)
class DCADecision:
    should_buy:         bool
    symbol:             str               # "BTC/USD" or "ETH/USD"
    amount_usd:         float             # 0.0 if should_buy is False
    reason:             str

def evaluate_dca(
    symbol:             str,
    fng_score:          float,            # 0–100
    btc_pct_above_200ma: float,           # e.g. 0.52 = 52% above 200MA
    last_buy_ts:        float,            # Unix timestamp of most recent DCA buy for this symbol
    now_ts:             float,
    base_amount_usd:    float = 4.00,
    interval_hours:     float = 168.0,    # Weekly
) -> DCADecision: ...
```

### Logic (exact)

1. **Interval gate:** if `now_ts - last_buy_ts < interval_hours * 3600` → `should_buy=False, reason="Interval not elapsed"`.
2. **Top filter:** if `btc_pct_above_200ma > 0.50` AND `fng_score >= 25` → skip. (Don't top-buy in euphoria, but fear overrides the filter — if everyone's panicking above the 200MA, keep buying.)
3. **F&G multiplier:**
   - `fng_score < 15` → `multiplier = 3.0`
   - `15 <= fng_score < 25` → `multiplier = 2.0`
   - else → `multiplier = 1.0`
4. `amount_usd = base_amount_usd * multiplier`
5. Enforce Kraken minimum: if `amount_usd < 3.80` → skip with reason.

### Tests required (test_dca.py)

- Interval not elapsed → should_buy=False
- F&G=10, interval elapsed, BTC +20% above 200MA → amount_usd=12.00
- F&G=20, interval elapsed → amount_usd=8.00
- F&G=50, BTC +80% above 200MA → skip (top filter)
- F&G=10, BTC +80% above 200MA → BUY (fear overrides)
- base=1.00, F&G=50 → skip (below Kraken minimum)

## 6. Satellite — Mean-Reversion Agent (agents/mean_reversion.py)

**Behavior:** BTC/USD only. Buys when BTC is oversold on 4h (RSI<35 + lower Bollinger Band touch + green confirmation candle) AND BTC is above its 200-day MA.

### Signal logic (exact)

Only returns `action="buy"` when ALL of:
1. `ctx.open_satellite_position is None` (no open position)
2. `ctx.btc_above_200ma is True` (trend filter)
3. RSI(14) on 4h <= 35.0
4. Last 4h close <= lower Bollinger Band (20, 2σ)
5. Current 4h candle (still forming) is green AND its low is above the prior candle's low (confirmation of reversal)

Returns `action="sell"` when `ctx.open_satellite_position` exists AND any of:
- `(current_price - entry_price) / entry_price >= target_pct` (take profit)
- `(current_price - entry_price) / entry_price <= -stop_pct` (stop loss)
- `(now_ts - entry_ts) / 3600 >= SAT_MAX_HOLD_HOURS` (time stop)

Else: `action="hold"`.

### Fixed parameters (constants in the file)

```python
SAT_RSI_THRESHOLD = 35.0
SAT_RSI_PERIOD = 14
SAT_RSI_TIMEFRAME = "4h"
SAT_BB_PERIOD = 20
SAT_BB_STD = 2.0
SAT_BB_TIMEFRAME = "4h"
SAT_TP_PCT = 0.08
SAT_SL_PCT = 0.04
SAT_MAX_HOLD_HOURS = 72.0
```

### AgentSignal fields to return

- `target_pct = 0.08`, `stop_pct = 0.04` (always, for this agent)
- `confidence` = simple average of 3 boolean conditions met / 3 (for telemetry; not used for sizing)
- `reason` = human-readable summary of which conditions fired
- `expected_hold_hours` = 36.0 (midpoint estimate)

### Tests required (test_mean_reversion.py)

Mock the exchange's `fetch_ohlcv`. Cover:
- BTC below 200MA → hold (trend filter)
- Position open + price hit +8% → sell (take profit)
- Position open + price hit −4% → sell (stop loss)
- Position open + 72h elapsed → sell (time stop)
- No position + RSI=40 → hold (not oversold enough)
- No position + RSI=30 + no BB touch → hold
- No position + RSI=30 + BB touch + red confirmation candle → hold
- No position + all conditions met → buy
- Signal has target_pct=0.08, stop_pct=0.04 on buy

## 7. Capital Module (capital.py)

**Purpose:** Prevent the satellite from trading when the account is unhealthy. NOT used for sizing.

### Public API

```python
from dataclasses import dataclass

@dataclass(frozen=True)
class CapitalState:
    total_value:           float
    core_value:            float
    satellite_value:       float
    satellite_enabled:     bool        # False when below pause threshold
    alert_floor_breached:  bool        # True when below $50
    runway_months:         float       # (total - floor) / monthly_burn
    reason:                str

# Thresholds (module constants)
CAP_SAT_PAUSE_THRESHOLD = 70.0   # USD — satellite paused below this
CAP_ALERT_FLOOR = 50.0           # USD — Telegram alert below this
CAP_FLOOR = 20.0                 # USD — hard floor for runway calc
CAP_MONTHLY_BURN = 12.0          # USD — fixed

def compute_state(
    total_value:       float,
    core_fraction:     float = 0.70,
    sat_fraction:      float = 0.30,
) -> CapitalState: ...
```

### Logic

- `core_value = total_value * core_fraction`, `satellite_value = total_value * sat_fraction`.
- `satellite_enabled = total_value >= CAP_SAT_PAUSE_THRESHOLD`.
- `alert_floor_breached = total_value < CAP_ALERT_FLOOR`.
- `runway_months = max(0.0, (total_value - CAP_FLOOR) / CAP_MONTHLY_BURN)`.

### Tests required (test_capital.py)

- $90 → enabled=True, alert=False, runway=5.83
- $65 → enabled=False (paused)
- $45 → enabled=False, alert=True
- $20 → runway=0.0
- $15 → runway=0.0 (clamped)

## 8. Strategy Orchestrator (strategy.py)

**One function.** No classes, no regime enum, no tier enum. Just wiring.

```python
from dataclasses import dataclass
from agents.base import AgentContext, AgentSignal
from agents import mean_reversion
from capital import compute_state, CapitalState

@dataclass(frozen=True)
class OrchestratorDecision:
    core_decision:      DCADecision | None     # From core.dca.evaluate_dca — may be None if not due
    satellite_signal:   AgentSignal | None     # From mean_reversion.evaluate — may be None if disabled
    capital_state:      CapitalState
    notes:              list[str]              # Human-readable reasoning for Telegram

def decide(
    exchange,
    account_value:     float,
    fng_score:         float,
    dca_last_buy_ts:   dict[str, float],   # {"BTC/USD": ts, "ETH/USD": ts}
    open_sat_position: dict | None,
    now_ts:            float,
) -> OrchestratorDecision: ...
```

### Logic

1. Compute capital state.
2. For each core symbol (BTC/USD, ETH/USD), call `evaluate_dca`. Use the first one that returns `should_buy=True` (serialized to avoid double-spend). Return as `core_decision`.
3. If `capital_state.satellite_enabled`:
    - Fetch btc_above_200ma from the exchange (one call)
    - Build AgentContext, call `mean_reversion.evaluate`, return as `satellite_signal`.
4. Else: `satellite_signal = None`, add note "Satellite paused: account below $70".
5. Return OrchestratorDecision.

### Tests required (test_strategy.py)

- Account=$65 → satellite_signal is None, note mentions pause
- Account=$90, DCA due for BTC, F&G=20 → core_decision.should_buy=True, amount=$8
- Account=$90, no DCA due → core_decision.should_buy=False
- Account=$90, sat position open, price at target → satellite_signal.action="sell"
